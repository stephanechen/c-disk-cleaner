# C 盘安全分析与清理技能

## 核心原则

1. 数据安全第一：任何删除/移动操作前必须获得用户明确确认
2. 先分析后操作：必须先完成完整扫描报告，再按用户选择逐步执行
3. 可回退设计：迁移操作用复制加目录联接方式，保留原文件备份
4. 透明每一步：每步操作前告知用户将要做什么、影响范围、预估耗时

## 工作流程

### 阶段一：扫描分析

使用 scripts/scan-disk.ps1 扫描 C 盘，生成完整的占用报告。

报告覆盖：C盘根目录系统文件、用户目录顶层、AppData、Program Files、ProgramData、Windows关键子目录。

### 阶段二：确认清理项

将报告展示给用户，请用户选择要清理的项目。每项标注安全等级和清理方式。

### 阶段三：逐步执行

每次只执行一项操作，完成一项后报告结果再进入下一项。

迁移流程：检查目标盘->关闭应用->robocopy复制->验证完整性->备份原目录->创建目录联接->验证联接。

清理流程：确认应用关闭->展示删除路径和大小->用户确认->执行删除。

### 阶段四：收尾报告

汇总空间变化和剩余建议。

## 安全等级

- 安全：纯缓存/临时文件（npm cache, pip cache, TEMP, 浏览器缓存）可脚本批量清理
- 需确认：用户数据（微信文件, WPS云数据, 剪映草稿）需迁移非直接删除
- 系统：系统文件（WinSxS, hiberfil.sys）用系统工具处理

## 不可做的事

- 绝不未经用户确认删除任何文件
- 绝不删除 Windows 系统文件
- 绝不在应用运行时操作其数据目录
- 绝不跳过备份步骤
