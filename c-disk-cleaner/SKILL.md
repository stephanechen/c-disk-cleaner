# C 盘安全分析与清理技能

## 架构总览

本技能采用三层可扩展架构，适配任意电脑环境：

```
SKILL.md              ← 工作流程 + 安全规则
scripts/
  scan-disk.ps1       ← 通用扫描引擎（不写死任何软件）
  migrate-data.ps1    ← 安全迁移引擎（复制→验证→联接→备份）
  safe-clean.ps1      ← 安全清理引擎（参数驱动）
references/
  software-registry.md ← 50+常见软件数据路径注册表（核心知识库）
  safe-cleanup-guide.md ← 安全清理知识库与操作原则
  custom-config-template.md ← 用户自定义软件路径模板
```

**核心设计思想：** 脚本只做通用扫描/迁移/清理逻辑，软件识别全部来自 `references/software-registry.md` 知识库。换一台电脑、换一批软件，只需要扩充知识库，脚本不用改。

## 核心原则

1. **数据安全第一：** 任何删除/移动操作前必须获得用户明确确认
2. **先分析后操作：** 必须先完成完整扫描报告，再按用户选择逐步执行
3. **可回退设计：** 迁移操作用"复制 + 目录联接"方式，保留 .bak 备份
4. **透明每一步：** 每步操作前告知用户将要做什么、影响范围、预估耗时
5. **跨机器适配：** 扫描引擎不硬编码软件路径，通过软件注册表知识库动态识别
6. **多用户支持：** 扫描所有 `C:\Users\*` 目录，不限于当前用户

## 工作流程

### 阶段一：扫描分析

运行 `scripts/scan-disk.ps1` 扫描 C 盘。脚本工作方式：

1. 遍历 C 盘关键位置（多用户目录、AppData、Program Files、ProgramData、Windows 特殊目录）
2. 对每个超过阈值的目录，在**软件注册表**中查找匹配（`references/software-registry.md`）
3. 匹配成功 → 标注软件名、安全等级、推荐操作
4. 匹配失败 → 标注"未识别大型目录"，标记为 🟡 需确认
5. 生成排序报告，分为三组：🟢安全可清理 / 🟡需确认 / 🔴系统文件

报告以表格呈现，每项含有：软件名、路径、大小、安全等级、建议操作、备注。

### 阶段二：确认清理项

将报告展示给用户（按安全等级分组），请用户逐项选择：

- **🟢 安全项：** 可批量勾选，一键清理
- **🟡 需确认项：** 逐一确认，每项说明迁移方案
- **🔴 系统项：** 仅告知，推荐用系统自带工具（磁盘清理、powercfg 等）

对于报告中的"未识别"项，询问用户其用途，并提示可加入自定义配置。

### 阶段三：逐步执行

每次只执行一项操作，完成一项后报告结果再进入下一项。

#### 迁移类操作（用 migrate-data.ps1）

1. 检查目标盘是否存在且有足够空间
2. 确认源应用已关闭（检查进程）
3. robocopy 复制数据到目标盘
4. 验证复制完整性（文件数 + 总大小对比）
5. 重命名源目录为 .bak 备份
6. 创建目录联接 `mklink /J`
7. 验证联接生效
8. 询问用户：是否删除 .bak 备份？

#### 清理类操作（用 safe-clean.ps1）

1. 确认目标应用已关闭
2. 展示将要删除的具体路径、内容说明、大小
3. 用户确认后执行删除
4. 报告释放空间

### 阶段四：收尾报告

- 每项操作释放/迁移的空间
- C 盘前后可用空间对比
- 遗留可清理项（含未识别目录的后续建议）

## 安全等级

| 等级 | 含义 | 示例 | 操作方式 |
|------|------|------|----------|
| 🟢 安全 | 纯缓存/临时文件 | npm cache, pip cache, %TEMP%, 浏览器缓存 | 可批量清理 |
| 🟡 需确认 | 用户数据/大型软件数据 | 微信文件, WPS云数据, 剪映草稿, Steam | 迁移或选择性清理 |
| 🔴 系统 | 系统文件 | WinSxS, hiberfil.sys, pagefile.sys | 告知并推荐系统工具 |

## 适配其他用户电脑

当技能包被复制到新电脑上使用时：

1. **自动扫描：** 遍历所有用户目录，不依赖特定用户名
2. **知识库匹配：** 50+ 常见软件自动识别
3. **未识别目录：** 列出未匹配的大型目录，提示用户标记
4. **自定义扩展：** 用户可将本机特有软件路径写入 `custom-config-template.md` 格式，后续扫描生效

### 扩展知识库

若发现新的常见软件需要加入知识库，在 `references/software-registry.md` 中对应分类添加条目，并同步更新 `scripts/scan-disk.ps1` 中的 `$SoftwareRegistry` 数据结构。

## 不可做的事

- 绝不未经用户确认删除任何文件
- 绝不删除 Windows 系统文件
- 绝不在应用运行时操作其数据目录
- 绝不跳过备份步骤直接删除迁移源文件
- 绝不假设用户意图，所有选择必须明确确认
- 对所有用户目录一视同仁，不默认只处理当前用户
